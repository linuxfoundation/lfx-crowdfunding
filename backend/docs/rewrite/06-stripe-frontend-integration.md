# Stripe SetupIntent — Frontend Integration Guide

> **Audience:** Frontend engineers who are new to Stripe or new to this codebase.
> This guide explains every concept from scratch, shows exactly which files to create
> or modify, and ends with a manual testing checklist.

---

## Overview

CF uses Stripe's **SetupIntent** flow for card management. Cards are authenticated once at save-time (handling 3DS challenges where required) and stored as a `PaymentMethod` (`pm_xxx`) per user. Donations and subscriptions reuse the saved payment method rather than collecting card details on each transaction.

---

## Complete backend endpoint reference

All endpoints live under the Go service (proxied through Nuxt's `server/api/`).
All protected endpoints require a `Bearer` JWT — the BFF proxy (step 1 below) adds it
automatically from the user's HTTP-only session cookie.

### Card management (SetupIntent flow)

| # | Method | Path | Auth | What it does |
|---|--------|------|------|--------------|
| 1 | `POST` | `/v1/me/setup-intent` | JWT | Creates a Stripe SetupIntent. Returns `{ client_secret }`. |
| 2 | `POST` | `/v1/me/payment-method` | JWT | After Stripe.js confirms the SetupIntent, send the resulting `pm_xxx` here to attach it to the user's Stripe Customer and save it in the DB. Returns `CardDetails`. |
| 3 | `GET`  | `/v1/me/payment-account` | JWT | Returns the user's saved card info (last4, brand, expiry, and `payment_method_id`). Returns `404` if no card is saved. |
| 4 | `DELETE` | `/v1/me/payment-method` | JWT | Detaches the card from Stripe and clears it from the DB. Returns `204 No Content`. |

### Payments

| # | Method | Path | Auth | Headers | What it does |
|---|--------|------|------|---------|--------------|
| 5 | `POST` | `/v1/initiatives/{id}/donations` | JWT | `Idempotency-Key: <uuid>` | Creates a one-time donation using a saved `pm_xxx`. |
| 6 | `POST` | `/v1/initiatives/{id}/subscriptions` | JWT | `Idempotency-Key: <uuid>` | Creates a recurring subscription using a saved `pm_xxx`. |
| 7 | `DELETE` | `/v1/subscriptions/{id}` | JWT | — | Cancels an active subscription. Returns `204`. |
| 8 | `GET` | `/v1/me/subscriptions` | JWT | — | Lists the authenticated user's own subscriptions (paginated). |

### Response shapes

**`CardDetails`** (endpoints 2 and 3):
```json
{
  "payment_method_id": "pm_1234",
  "last_four": "4242",
  "brand": "visa",
  "expiry_month": 12,
  "expiry_year": 2027
}
```

**`Donation`** (endpoint 5) — `client_secret` only appears when 3DS is required:
```json
{
  "id": "uuid",
  "status": "succeeded",
  "client_secret": "pi_xxx_secret_yyy"
}
```

**`Subscription`** (endpoint 6) — `client_secret` only appears when first invoice needs 3DS:
```json
{
  "id": "uuid",
  "status": "incomplete",
  "client_secret": "pi_xxx_secret_yyy"
}
```

**`GET /v1/me/subscriptions`** (endpoint 8):
```json
{
  "data": [
    {
      "id": "uuid",
      "initiative_id": "uuid",
      "current_amount_in_cents": 5000,
      "frequency": "monthly",
      "status": "active"
    }
  ],
  "meta": { "total": 1, "limit": 20, "offset": 0 }
}
```

## The three flows explained simply

### Flow A — Save a card (SetupIntent)

```
User enters card details
        │
        ▼
Frontend calls POST /api/payment/setup-intent   (Nuxt BFF → Go backend)
        │
        ◄── { client_secret: "seti_xxx_secret_yyy" }
        │
        ▼
Stripe.js calls stripe.confirmCardSetup(client_secret, { card: cardElement })
        │   (This is where the 3DS challenge popup appears, if required)
        │
        ◄── { setupIntent: { payment_method: "pm_1RealCard" } }
        │
        ▼
Frontend calls POST /api/payment/method  body: { payment_method_id: "pm_1RealCard" }
        │
        ◄── CardDetails { last_four: "4242", ... }
        │
        ▼
Card is now saved. Frontend shows "Visa ending 4242".
```

### Flow B — One-time donation

```
User clicks "Donate $100"
        │
        ▼
Does the user have a saved card?
  GET /api/payment/account
  ├── 404 → run Flow A first, then come back here
  └── 200 → CardDetails.payment_method_id = "pm_1RealCard"
        │
        ▼
POST /api/initiatives/{id}/donations
  header:  Idempotency-Key: <new UUID generated by the frontend>
  body:    { amount_in_cents: 10000, stripe_payment_method_id: "pm_1RealCard" }
        │
        ◄── { status: "succeeded", client_secret: null }   ← most cards
        ◄── { status: "requires_action", client_secret: "pi_xxx_secret" }   ← 3DS bank
        │
        ▼  (only when client_secret is present)
stripe.confirmCardPayment(client_secret)
        │   (3DS popup — user taps "Approve" in their banking app)
        │
        ▼
Show success screen
```

### Flow C — Monthly subscription

```
Same as Flow B except:
  • Must have a saved card (subscriptions always require a pm_xxx)
  • POST /api/initiatives/{id}/subscriptions  { frequency: "monthly", ... }
  • If client_secret present, call stripe.confirmCardPayment(client_secret)
    (the backend already set the payment method on the invoice's PaymentIntent;
     confirmCardPayment handles the 3DS challenge inline without a page redirect)
```

## Implementation steps

### Step 1 — BFF proxy utility

Every call to the Go backend needs the user's JWT attached. Create a shared server
utility so all proxy routes do this automatically.

**`frontend/server/utils/backend-fetch.ts`**
```ts
// Copyright The Linux Foundation and each contributor to LFX.
// SPDX-License-Identifier: MIT
import type { H3Event } from 'h3'
import { getHeader } from 'h3'

/**
 * Forwards a request to the Go backend, attaching the user's JWT from the
 * Authorization header that the Nuxt auth middleware already validates.
 *
 * Usage inside a server route:
 *   return useBackendFetch(event, '/v1/me/setup-intent', { method: 'POST' })
 */
export const useBackendFetch = async <T = unknown>(
  event: H3Event,
  path: string,
  options: {
    method?: 'GET' | 'POST' | 'DELETE' | 'PATCH'
    body?: unknown
    headers?: Record<string, string>
  } = {},
): Promise<T> => {
  const config = useRuntimeConfig()
  const baseURL = config.backendBaseUrl as string

  // Forward the JWT the client sent us
  const authorization = getHeader(event, 'authorization') ?? ''

  return $fetch<T>(path, {
    baseURL,
    method: options.method ?? 'GET',
    body: options.body,
    headers: {
      ...(authorization ? { authorization } : {}),
      ...options.headers,
    },
  })
}
```

Add `backendBaseUrl` to `setup/runtime-config.ts`:

```ts
// In the default export object, server-only block:
backendBaseUrl: process.env.NUXT_BACKEND_BASE_URL || 'http://localhost:8080',
```

Add to `.env`:
```
NUXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_test_...
NUXT_BACKEND_BASE_URL=http://localhost:8080
```

### Step 4 — `usePaymentAccount` composable

This composable owns all card-management state. It uses `useState` so the saved
card is shared across components (e.g. header, donate drawer, account page).

**`frontend/app/composables/usePaymentAccount.ts`**

```ts
// Copyright The Linux Foundation and each contributor to LFX.
// SPDX-License-Identifier: MIT
import type { StripeCardNumberElement } from '@stripe/stripe-js'
import type { CardDetails, SetupIntentResult } from '#shared/types/payment.types'

/**
 * Manages the user's saved Stripe card.
 *
 * State is shared via useState('payment-account') so any component can read
 * `card` without re-fetching.
 *
 * Example:
 *   const { card, fetchCard, saveCard, removeCard } = usePaymentAccount()
 *   await fetchCard()            // call once on page load
 *   await saveCard(cardElement)  // full SetupIntent flow
 */
export const usePaymentAccount = () => {
  // Shared state — all components that call usePaymentAccount() share this ref.
  const card = useState<CardDetails | null>('payment-account', () => null)
  const loading = ref(false)
  const error = ref<string | null>(null)

  /**
   * Loads the user's saved card from the backend.
   * Sets card to null if none is saved (404 is not treated as an error).
   */
  const fetchCard = async () => {
    loading.value = true
    error.value = null
    try {
      card.value = await $fetch<CardDetails>('/api/payment/account')
    } catch (e: any) {
      if (e?.statusCode === 404) {
        // No card on file — this is normal for new users.
        card.value = null
      } else {
        error.value = e?.data?.message ?? 'Could not load your payment account.'
      }
    } finally {
      loading.value = false
    }
  }

  /**
   * Full SetupIntent card-save flow:
   *  1. Ask the backend to create a SetupIntent → get client_secret
   *  2. Tell Stripe.js to confirm the card using that secret (3DS challenge if needed)
   *  3. Send the resulting pm_xxx to the backend to save it
   *
   * @param cardElement  The mounted StripeCardNumberElement from donate-step-payment.vue
   * @throws             Re-throws on any failure so the caller can show an error
   */
  const saveCard = async (cardElement: StripeCardNumberElement) => {
    const { getStripe } = useStripe()
    loading.value = true
    error.value = null

    try {
      // Step 1 — create the SetupIntent on the backend
      const { client_secret } = await $fetch<SetupIntentResult>('/api/payment/setup-intent', {
        method: 'POST',
      })

      // Step 2 — confirm the card with Stripe.js
      //   stripe.confirmCardSetup authenticates the card and, if the user's
      //   bank requires it, shows a 3DS popup. On success we get the pm_xxx.
      const stripe = await getStripe()
      if (!stripe) throw new Error('Stripe.js failed to load — check NUXT_PUBLIC_STRIPE_PUBLISHABLE_KEY.')

      const { setupIntent, error: stripeError } = await stripe.confirmCardSetup(client_secret, {
        payment_method: { card: cardElement },
      })

      if (stripeError) {
        // Stripe returns human-readable messages (e.g. "Your card was declined.")
        throw new Error(stripeError.message)
      }
      if (!setupIntent?.payment_method) {
        throw new Error('Stripe did not return a payment method. Please try again.')
      }

      // Step 3 — persist pm_xxx to the backend
      const paymentMethodId = typeof setupIntent.payment_method === 'string'
        ? setupIntent.payment_method
        : setupIntent.payment_method.id

      card.value = await $fetch<CardDetails>('/api/payment/method', {
        method: 'POST',
        body: { payment_method_id: paymentMethodId },
      })
    } catch (e: any) {
      error.value = e?.message ?? 'Failed to save your card.'
      throw e
    } finally {
      loading.value = false
    }
  }

  /**
   * Removes the user's saved card from Stripe and the database.
   */
  const removeCard = async () => {
    loading.value = true
    error.value = null
    try {
      await $fetch('/api/payment/method', { method: 'DELETE' })
      card.value = null
    } catch (e: any) {
      error.value = e?.data?.message ?? 'Failed to remove your card.'
      throw e
    } finally {
      loading.value = false
    }
  }

  return { card, loading, error, fetchCard, saveCard, removeCard }
}
```

### Step 5 — `useDonate` composable

**`frontend/app/composables/useDonate.ts`**

```ts
// Copyright The Linux Foundation and each contributor to LFX.
// SPDX-License-Identifier: MIT
import type { DonationResult } from '#shared/types/payment.types'

export interface DonateInput {
  amount_in_cents: number
  stripe_payment_method_id: string  // pm_xxx from card.payment_method_id
  category?: string
  organization_id?: string
}

/**
 * Submits a one-time donation.
 *
 * The caller must already have a saved card. Retrieve payment_method_id via:
 *   const { card } = usePaymentAccount()
 *
 * This composable generates a fresh idempotency key per call. That means if
 * the user taps "Donate" twice by mistake, the second tap creates a new key
 * and a new charge. To prevent double-charging, disable the button while
 * loading.value is true.
 *
 * Example:
 *   const { loading, donate } = useDonate()
 *   await donate(initiative.id, { amount_in_cents: 5000, stripe_payment_method_id: card.value.payment_method_id })
 */
export const useDonate = () => {
  const loading = ref(false)
  const error = ref<string | null>(null)

  const donate = async (initiativeId: string, input: DonateInput): Promise<DonationResult> => {
    const { getStripe } = useStripe()
    loading.value = true
    error.value = null

    try {
      const result = await $fetch<DonationResult>(`/api/initiatives/${initiativeId}/donations`, {
        method: 'POST',
        body: input,
        // A UUID generated here ensures that if the network drops and the
        // browser retries the request, Stripe won't create a duplicate charge.
        headers: { 'Idempotency-Key': crypto.randomUUID() },
      })

      // 3DS branch — only reached if the user's bank required a challenge.
      // Status is "requires_action" and client_secret is non-empty.
      if (result.client_secret) {
        const stripe = await getStripe()
        if (!stripe) throw new Error('Stripe.js failed to load.')

        const { error: stripeError } = await stripe.confirmCardPayment(result.client_secret)
        if (stripeError) throw new Error(stripeError.message)
      }

      return result
    } catch (e: any) {
      error.value = e?.data?.message ?? e?.message ?? 'Donation failed. Please try again.'
      throw e
    } finally {
      loading.value = false
    }
  }

  return { loading, error, donate }
}
```

### Step 6 — `useSubscribe` composable

**`frontend/app/composables/useSubscribe.ts`**

```ts
// Copyright The Linux Foundation and each contributor to LFX.
// SPDX-License-Identifier: MIT
import type { SubscriptionResult, UserSubscription, PaginationMeta } from '#shared/types/payment.types'

export interface SubscribeInput {
  amount_in_cents: number
  frequency: 'monthly' | 'yearly'
  stripe_payment_method_id: string  // pm_xxx from card.payment_method_id
  category?: string
  organization_id?: string
}

/**
 * Creates and manages recurring subscriptions.
 *
 * Example:
 *   const { loading, subscribe, cancel, mySubscriptions, fetchMySubscriptions } = useSubscribe()
 *   await subscribe(initiative.id, { amount_in_cents: 2000, frequency: 'monthly', stripe_payment_method_id: card.value.payment_method_id })
 */
export const useSubscribe = () => {
  const loading = ref(false)
  const error = ref<string | null>(null)
  const mySubscriptions = ref<UserSubscription[]>([])
  const meta = ref<PaginationMeta | null>(null)

  /**
   * Creates a recurring subscription.
   * When the first invoice requires 3DS, uses redirect: 'if_required' so the
   * challenge is handled inline rather than redirecting the whole page.
   */
  const subscribe = async (
    initiativeId: string,
    input: SubscribeInput,
  ): Promise<SubscriptionResult> => {
    const { getStripe } = useStripe()
    loading.value = true
    error.value = null

    try {
      const result = await $fetch<SubscriptionResult>(
        `/api/initiatives/${initiativeId}/subscriptions`,
        {
          method: 'POST',
          body: input,
          headers: { 'Idempotency-Key': crypto.randomUUID() },
        },
      )

      // 3DS branch — first invoice needs authentication.
      // Status is "incomplete" until the challenge is completed.
      // The backend already attached the payment method to the invoice's PaymentIntent
      // (via DefaultPaymentMethod on the subscription), so we only need the client_secret.
      // Use confirmCardPayment (the split card element API), NOT confirmPayment (Payment
      // Element API) — they look similar but confirmPayment requires a mounted Elements
      // instance and would throw "You must provide an `elements` instance".
      if (result.client_secret) {
        const stripe = await getStripe()
        if (!stripe) throw new Error('Stripe.js failed to load.')

        const { error: stripeError } = await stripe.confirmCardPayment(result.client_secret)
        if (stripeError) throw new Error(stripeError.message)
      }

      return result
    } catch (e: any) {
      error.value = e?.data?.message ?? e?.message ?? 'Subscription failed. Please try again.'
      throw e
    } finally {
      loading.value = false
    }
  }

  /**
   * Cancels an active subscription.
   * @param subscriptionId  The UUID from UserSubscription.id (not the Stripe sub ID)
   */
  const cancel = async (subscriptionId: string): Promise<void> => {
    loading.value = true
    error.value = null
    try {
      await $fetch(`/api/subscriptions/${subscriptionId}`, { method: 'DELETE' })
      // Remove from local state immediately for instant UI feedback
      mySubscriptions.value = mySubscriptions.value.filter((s) => s.id !== subscriptionId)
    } catch (e: any) {
      error.value = e?.data?.message ?? e?.message ?? 'Could not cancel subscription.'
      throw e
    } finally {
      loading.value = false
    }
  }

  /**
   * Loads the current user's subscriptions.
   * Useful for an account page that lists "your monthly donations".
   */
  const fetchMySubscriptions = async (limit = 20, offset = 0): Promise<void> => {
    loading.value = true
    error.value = null
    try {
      const response = await $fetch<{ data: UserSubscription[]; meta: PaginationMeta }>(
        `/api/me/subscriptions?limit=${limit}&offset=${offset}`,
      )
      mySubscriptions.value = response.data
      meta.value = response.meta
    } catch (e: any) {
      error.value = e?.data?.message ?? 'Could not load your subscriptions.'
    } finally {
      loading.value = false
    }
  }

  return { loading, error, subscribe, cancel, mySubscriptions, meta, fetchMySubscriptions }
}
```

### Step 7 — Update `donate-step-payment.vue`

The current component exposes `createPaymentMethod()` which calls
`stripe.createPaymentMethod()` — the old approach that skips 3DS. Replace it with
`confirmSetup()` which runs the full SetupIntent flow.

**Only the `<script setup>` section needs to change.** The template (the card
number/expiry/cvc inputs) stays exactly the same.

`donate-step-payment.vue` exposes `confirmSetup` and `cardNumberElement`:

```ts
// Uses SetupIntent so the card is 3DS-authenticated at save-time.
// The parent (donate-drawer.vue) calls confirmSetup() and gets back
// the pm_xxx that was authenticated by Stripe.
const confirmSetup = async (): Promise<string> => {
  const stripe = await getStripe();
  if (!stripe || !cardNumberEl) throw new Error('Stripe not loaded');

  stripeError.value = '';

  // Step 1 — create the SetupIntent
  let clientSecret: string;
  try {
    const result = await $fetch<{ client_secret: string }>('/api/payment/setup-intent', {
      method: 'POST',
    });
    clientSecret = result.client_secret;
  } catch {
    stripeError.value = 'Could not initialise payment. Please try again.';
    throw new Error(stripeError.value);
  }

  // Step 2 — confirm with Stripe.js (3DS challenge runs here if required)
  const { setupIntent, error } = await stripe.confirmCardSetup(clientSecret, {
    payment_method: { card: cardNumberEl },
  });

  if (error) {
    stripeError.value = error.message ?? 'Card declined. Please try a different card.';
    throw new Error(error.message);
  }
  if (!setupIntent?.payment_method) {
    stripeError.value = 'Unexpected error. Please try again.';
    throw new Error(stripeError.value);
  }

  return typeof setupIntent.payment_method === 'string'
    ? setupIntent.payment_method
    : setupIntent.payment_method.id;
};

defineExpose({ confirmSetup });
```

Inside `handleContinue`, the `isLastStep` branch:

// Check for existing saved card; only run SetupIntent if none exists
const { card, fetchCard, saveCard } = usePaymentAccount()

// Ensure card state is loaded before the last step renders
onMounted(() => fetchCard())

// Inside handleContinue, isLastStep branch:
submitting.value = true;
try {
  // Step 1 — resolve the payment method ID:
  //   • Returning user: they already have a saved card → use it directly.
  //     No new SetupIntent is created; the existing card is charged.
  //   • New user: run the full SetupIntent + 3DS flow to save a card first.
  let paymentMethodId: string
  if (card.value?.payment_method_id) {
    // Existing card — skip the SetupIntent step entirely.
    paymentMethodId = card.value.payment_method_id
  } else {
    // New card — confirmSetup runs the SetupIntent (3DS challenge if needed)
    // and returns the pm_xxx that was authenticated by Stripe.
    const cardElement = paymentStepRef.value?.cardNumberElement
    if (!cardElement) throw new Error('Card element not mounted')
    await saveCard(cardElement)                  // saves card.value automatically
    paymentMethodId = card.value!.payment_method_id
  }

  // Step 2 — create the donation using the saved card
  await donate(props.initiative.id, {
    amount_in_cents: amountForm.value.amountCents,
    stripe_payment_method_id: paymentMethodId,
  });

  submitted.value = true;
  emit('submitted');
} catch {
  // Errors are already displayed inside donate-step-payment (stripeError)
  // or by useDonate/usePaymentAccount (error.value). No extra handling needed.
} finally {
  submitting.value = false;
}

## Error handling reference

| Situation | What happens | User sees |
|---|---|---|
| Stripe.js fails to load | `getStripe()` returns `null` | "Stripe.js failed to load — check NUXT_PUBLIC_STRIPE_PUBLISHABLE_KEY." |
| Card declined at SetupIntent | `stripe.confirmCardSetup` → `error.message` | "Your card was declined." (Stripe's own message) |
| 3DS challenge failed | Same as above | "We are unable to authenticate your payment method." |
| 3DS challenge closed by user | Same as above | "Your card was not authenticated, please try again." |
| Backend 400 (invalid input) | `$fetch` throws with `statusCode: 400` | `e.data.message` from backend |
| Backend 503 (Stripe down) | `$fetch` throws with `statusCode: 503` | `e.data.message` from backend |
| Network timeout + retry | Idempotency-Key prevents double-charge | User retries safely; Stripe returns the same result |
| Double-tap "Donate" button | Button disabled while `loading.value === true` | Second tap is silently ignored |

## Manual testing checklist

Use Stripe's test card numbers from https://stripe.com/docs/testing.

### Save a card (Flow A)
- [ ] Enter `4242 4242 4242 4242` exp: any future date, CVC: any 3 digits

- [ ] `GET /api/payment/account` returns `{ payment_method_id: "pm_xxx", last_four: "4242" }`

- [ ] Enter `4000 0025 0000 3155` (always triggers 3DS)
- [ ] A Stripe popup appears — click "Complete"
- [ ] Card saves successfully
- [ ] Enter same card again — click "Fail" in the popup
- [ ] Error message appears: "We are unable to authenticate…"

- [ ] Donating again shows the card-entry form

- [ ] Close the 3DS popup without completing → error shown, card NOT saved, user can retry
- [ ] Two taps on "Donate" button while submitting → button is disabled, only one charge created
